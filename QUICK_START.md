# Bordo - Краткое руководство

## Начало работы

### 1. Убедитесь что запущен Spring Backend
- Backend должен быть доступен по адресу: `http://localhost:8080`
- API endpoints должны быть реализованы согласно документации

### 2. Запустите приложение
```bash
cd college
npm install  # если еще не установили
npm run dev
```

### 3. Откройте в браузере
```
http://localhost:5173
```

## Основные страницы приложения

### 🔐 Страница входа (`/login`)
- Введите email и пароль
- Ссылка на регистрацию для новых пользователей
- Токен автоматически сохраняется в localStorage

### 📝 Регистрация (`/register`)
- Заполните форму с личными данными
- Пароль: 5-8 символов
- Email должен быть действительным
- После регистрации автоматически логинитесь

### 🏠 Главная страница - Каталог (`/`)
- Все активные объявления в виде сетки
- Слева: фильтрация по категориям
- Клик на карточку → открыть детали объявления
- Доступна для всех, включая неавторизованных

### 🔍 Детали объявления (`/ads/:id`)
- Полная информация о скоте
- Контакты продавца
- Кнопка "Добавить в избранное" (только для авторизованных)
- История просмотров
- Информация о местоположении

### ❤️ Избранное (`/favorites`)
- Просмотр всех сохраненных объявлений
- Удаление из избранного
- Быстрый переход к полной информации

### 👤 Профиль (`/profile`)

**Вкладка "Личная информация":**
- ФИО, Email, Телефон
- Дата регистрации
- Статистика объявлений

**Вкладка "Мои объявления":**
- Список всех ваших объявлений
- Статус каждого (ACTIVE, SOLD, INACTIVE)
- Кнопки редактирования и удаления

### ➕ Создание объявления (`/ads/create`)
- Название (обязательно)
- Описание (обязательно)
- Цена (обязательно)
- Категория (обязательно)
- Локация (опционально)
- Валюта (RUB, USD, EUR)

## Часто встречающиеся вопросы

**Q: Я забыл пароль, что делать?**
A: В текущей версии нет функции восстановления пароля. Создайте новый аккаунт.

**Q: Как удалить объявление?**
A: Перейдите в "Профиль" → "Мои объявления" → кнопка "Удалить"

**Q: Как связаться с продавцом?**
A: Откройте детали объявления и нажмите "Связаться с продавцом"

**Q: Моя ошибка аутентификации 401**
A: Проверьте правильность данных входа или зарегистрируйтесь заново

**Q: Почему не загружаются объявления?**
A: Убедитесь что:
1. Spring Backend запущен на `http://localhost:8080`
2. API endpoints реализованы корректно
3. CORS включен на сервере

## Технические детали

### API базовый URL
```
http://localhost:8080/api
```

### Аутентификация
- Используется Bearer token (JWT)
- Автоматически добавляется в заголовок `Authorization`
- Хранится в `localStorage` под ключом `authToken`

### Сохраняемые данные в браузере
```javascript
localStorage.authToken  // JWT токен
```

### CORS требования на backend
```javascript
Access-Control-Allow-Origin: http://localhost:5173
Access-Control-Allow-Credentials: true
Access-Control-Allow-Methods: GET, POST, PUT, DELETE, OPTIONS
Access-Control-Allow-Headers: Content-Type, Authorization
```

## Развертывание

### Production сборка
```bash
npm run build
```
Результат: папка `dist/` готова к деплою

### Preview production сборки локально
```bash
npm run build
npm run preview
```

### Деплой на сервер
1. Выполните `npm run build`
2. Загрузите содержимое папки `dist/` на веб-сервер
3. Убедитесь что backend API доступен
